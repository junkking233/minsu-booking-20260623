## 发现的问题

### 高危
1. **订单号生成重复**：`generateOrderNo()` 方法在订单已存在时会生成重复订单号，导致 `DataIntegrityViolationException`
   - 原因：种子数据或测试数据中存在当天的订单记录
   - 修复方向：订单号生成需要更健壮（加锁或UUID）

### 中危
2. **不存在的接口返回 500 而不是 404**
   - 例如：`/api/admin/users`（正确是 `/api/users`）、`/api/admin/dashboard`（正确是 `/api/admin/dashboard/overview`）
   - 修复方向：配置 Spring Boot 的 404 错误处理

### 低危
3. **前端 API 调用与后端接口可能存在不匹配**
   - 需要逐个页面检查前端 API 调用是否与后端接口一致
   - 例如：ReviewController 没有 `/my` 端点，但前端可能没有调用它（需要确认）
