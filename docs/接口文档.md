# 民宿预约管理系统 · 接口文档

> **版本：** v1.0  
> **日期：** 2026-06-23  
> **后端端口：** 8888  
> **前端端口：** 9999  
> **Base Path：** `/api`

---

## 通用约定

### 统一响应结构

```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "timestamp": 1706603943000
}
```

### 错误响应

```json
{
  "code": 4001,
  "message": "具体错误信息",
  "data": null,
  "timestamp": 1706603943000
}
```

### 通用错误码

| Code | 含义 |
| --- | --- |
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未登录或登录已过期 |
| 403 | 无权限或账号已被禁用 |
| 404 | 资源不存在 |
| 409 | 数据冲突（唯一约束冲突） |
| 500 | 服务器内部错误 |

### 鉴权方式

- 请求头：`Authorization: Bearer {token}`
- 登录接口和注册接口不需要鉴权
- 其他接口均需鉴权

### 分页约定

- 分页参数：`page`（页码，从1开始）、`pageSize`（每页条数，默认20）
- 分页响应结构：

```json
{
  "records": [],
  "total": 100,
  "size": 20,
  "current": 1,
  "pages": 5
}
```

### 角色说明

| 角色 | 说明 |
| --- | --- |
| ADMIN | 后台管理员，单一账号 |
| USER | 普通用户（旅客） |

### 订单状态枚举

| 状态值 | 中文名 | 说明 |
| --- | --- | --- |
| PENDING | 待确认 | 用户提交预约后 |
| WAIT_PAY | 待支付 | 管理员确认后 |
| BOOKED | 已预订 | 用户支付后 |
| CHECKED_IN | 入住中 | 入住日期当天系统自动流转 |
| COMPLETED | 已完成 | 管理员标记完成 |
| REVIEWED | 已评价 | 用户发表评价后 |
| CANCELLED | 已取消 | 用户取消/管理员拒绝/超时自动取消 |
| REFUNDING | 退款中 | 管理员发起退款 |
| REFUNDED | 已退款 | 管理员同意退款 |

## 一、认证模块

### 1.1 用户注册

- Path: `/api/auth/register`
- Method: `POST`
- Roles: 无需鉴权

#### 请求参数

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| username | string | yes | 用户名（3-20位） |
| password | string | yes | 密码（6-20位） |
| name | string | no | 昵称，默认与用户名相同 |

#### 请求示例

```json
{
  "username": "zhangsan",
  "password": "123456",
  "name": "张三"
}
```

#### 成功响应

```json
{
  "code": 200,
  "message": "注册成功",
  "data": {
    "id": 4,
    "username": "zhangsan",
    "name": "张三",
    "role": "USER",
    "status": 1
  },
  "timestamp": 1706603943000
}
```

#### 错误场景

| Code | Scenario | Message |
| --- | --- | --- |
| 400 | 用户名为空或长度不合法 | 用户名必须为3-20位 |
| 400 | 密码为空或长度不合法 | 密码必须为6-20位 |
| 409 | 用户名已存在 | 用户名已被注册 |

## 五、订单模块

### 5.1 创建订单

- Path: `/api/orders`
- Method: `POST`
- Roles: USER

#### 请求参数

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| houseId | integer | yes | 房源ID |
| checkIn | string | yes | 入住日期 |
| checkOut | string | yes | 离店日期 |
| guests | integer | yes | 入住人数 |
| guestName | string | yes | 联系人姓名 |
| phone | string | yes | 联系人手机号 |
| note | string | no | 备注 |

#### 成功响应

```json
{
  "code": 200,
  "message": "预约成功",
  "data": {
    "orderNo": "MS202606230001",
    "status": "PENDING",
    "amount": 818.00,
    "priceDetail": {
      "roomFee": 798.00,
      "cleanFee": 50.00,
      "discount": 30.00,
      "total": 818.00
    }
  },
  "timestamp": 1706603943000
}
```

#### 错误场景

| Code | Scenario | Message |
| --- | --- | --- |
| 400 | 姓名为空 | 请填写联系人姓名 |
| 400 | 手机号格式不正确 | 手机号格式不正确 |
| 400 | 日期不合法 | 离店日期必须晚于入住日期 |
| 400 | 人数超限 | 入住人数超过最大可住人数 |
| 400 | 房源不可预约 | 该房源在所选日期不可预约 |
| 404 | 房源不存在 | 房源不存在或已下架 |

#### Notes
- 订单编号格式：MS + yyyyMMdd + 4位序号
- 金额计算：price × nights + cleanFee - discount（discount 来自系统配置）
- 创建后状态为 PENDING（待确认）

## 状态流转约束

```
PENDING --confirm--> WAIT_PAY --pay--> BOOKED --auto(timed)--> CHECKED_IN --complete--> COMPLETED --review--> REVIEWED
PENDING --reject--> CANCELLED
PENDING --cancel(user)--> CANCELLED
WAIT_PAY --cancel(user)--> CANCELLED
WAIT_PAY --timeout(auto)--> CANCELLED
BOOKED --cancel(user)--> CANCELLED
BOOKED --refund(admin)--> REFUNDING --approve--> REFUNDED
                                   --reject--> BOOKED (回退)
```

### 定时任务

1. **待支付超时取消**：扫描 WAIT_PAY 状态且创建时间超过配置的 `payTimeoutMinutes`（默认30分钟）的订单，自动变更为 CANCELLED
2. **入住自动流转**：扫描 BOOKED 状态且 checkIn 日期等于当天的订单，自动变更为 CHECKED_IN
